5) RoBERTa reduced experiments for laptop feasibility

Install the pinned Transformers version first:
  python3 -m pip install -r sentiment/requirements.txt

Start with this smoke test. It uses a tiny RoBERTa-like checkpoint to verify the
pipeline without loading roberta-base:

python3 sentiment/train.py --model roberta --transformer_model sshleifer/tiny-distilroberta-base --train sentiment/data/train.ft.txt --test sentiment/data/test.ft.txt --max_train 100 --max_test 100 --epochs 1 --max_len 64 --batch_size 2 --lr 2e-5 --num_layers_to_freeze -1 --output_dir sentiment/out/experiments/roberta_tiny_smoke --seed 42 --no_plots

If the smoke test works, run roberta-base with only the classification head trainable:

python3 sentiment/train.py --model roberta --transformer_model roberta-base --train sentiment/data/train.ft.txt --test sentiment/data/test.ft.txt --max_train 500 --max_test 500 --epochs 1 --max_len 64 --batch_size 2 --lr 2e-5 --num_layers_to_freeze -1 --output_dir sentiment/out/experiments/roberta_head_only_500 --seed 42 --no_plots

Then try partial fine-tuning by freezing the first 10 encoder layers:

python3 sentiment/train.py --model roberta --transformer_model roberta-base --train sentiment/data/train.ft.txt --test sentiment/data/test.ft.txt --max_train 1000 --max_test 500 --epochs 1 --max_len 128 --batch_size 4 --lr 2e-5 --num_layers_to_freeze 10 --output_dir sentiment/out/experiments/roberta_freeze_10_1k --seed 42 --no_plots

Only if the previous runs are stable, try full fine-tuning on a tiny subset:

python3 sentiment/train.py --model roberta --transformer_model roberta-base --train sentiment/data/train.ft.txt --test sentiment/data/test.ft.txt --max_train 1000 --max_test 500 --epochs 1 --max_len 128 --batch_size 2 --lr 2e-5 --num_layers_to_freeze 0 --output_dir sentiment/out/experiments/roberta_full_finetune_1k --seed 42 --no_plots

On Apple Silicon, you can try adding --device mps. If it crashes or becomes
unstable, remove it and use CPU with a smaller batch size.
